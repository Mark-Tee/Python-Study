当用户在对话中表现出对某个 Python 概念的混淆、不理解或频繁追问时，应主动定位 README.md 中对应的章节，针对性地补充说明、添加对比示例、增加图解注释或常见误区提示，帮助用户更快理解和掌握。

**触发条件：**
- 用户说"不太理解"、"有点混淆"、"这个我不懂"
- 用户反复追问同一个概念
- 用户的代码暴露出对某个知识点的误解

**做法：**
- 直接在 README.md 的对应章节中补充内容（新增小节、扩展示例、添加 ⚠️ 误区提示）
- 保持与现有笔记一致的格式风格
- 不修改章节编号（除非新增整节）
- 完成后告知用户在哪个章节做了补充

**Why:** 用户在学 Python，笔记是其最重要的参考资料。将澄清后的理解即时沉淀到笔记中，避免下次遇到同样问题再查。

**How to apply:** 聊天中注意到用户困惑 → 先回答问题 → 然后定位 README.md 对应章节 → Edit 补充内容 → 告知用户更新位置。
