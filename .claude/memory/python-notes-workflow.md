用户在学习 Python 课程，使用 `d:\桌面\PythonTest\README.md` 作为学习笔记。

**工作流：**
1. 用户上课时在 README.md 文件**末尾**快速追加粗略的新知识点（不关心格式）
2. 用户会要求 Claude 将这些粗略笔记整理成与现有内容一致的格式

**现有笔记格式规范：**
- 使用标准 Markdown + emoji（不用 HTML span 颜色标签）
- 章节标题：`## 序号. 主题名`，小节标题：`### 🔷 小节名`
- 重要提示用 `> ⚠️ **注意**` 引用块
- 最佳实践用 `> 💡 **推荐/最佳实践**` 引用块
- 大量使用结构化表格（对比表、速查表、方法一览表）
- 代码块都有 `# → 输出结果` 注释说明运行结果
- 每个小节都有实用代码示例
- 章节间用 `---` 分隔

**整理时需要做的：**
- 识别文件末尾新增的粗略内容
- 保持与现有章节相同的格式风格
- 自动接续章节编号
- 更新顶部目录
- 适当补充相关知识点、常见错误提示和实用技巧
- 保持"适合入门查阅和快速复习"的定位
